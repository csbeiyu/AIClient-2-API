# AIClient-2-API 部署教程

> 本文档介绍如何使用 1Panel 和 Docker 部署 AIClient-2-API 服务

---

## 📋 前置要求

- Docker 已安装并运行
- 1Panel 面板已安装（可选）
- 端口未被占用：3000, 8085-8087, 1455, 19876-19880

---

## 方式一：1Panel 构建镜像

### 步骤 1：打开构建界面

1. 登录 1Panel 面板
2. 进入「容器」→「镜像」
3. 点击「构建镜像」按钮

### 步骤 2：填写参数

| 字段 | 填写内容 | 说明 |
|------|----------|------|
| **名称** | `aiclient2api:latest` | 镜像名称和标签 |
| **Dockerfile** | 选择「路径选择」→ `Dockerfile` | Dockerfile 相对路径 |
| **构建上下文路径** | `/Users/zmy/Desktop/AIClient` | 项目根目录绝对路径 |
| **构建参数** | (留空) | 本项目不需要构建参数 |
| **标签** | `aiclient2api:latest` | 镜像标签 |

### 步骤 3：开始构建

1. 点击「构建」按钮
2. 等待构建完成（首次约 5-10 分钟）
3. 在镜像列表查看 `aiclient2api:latest`

### 步骤 4：创建容器

1. 进入「容器」→「容器」
2. 点击「创建容器」
3. 选择镜像：`aiclient2api:latest`
4. 容器名称：`aiclient2api`
5. **端口映射**：
   ```
   3000:3000
   8085:8085
   8086:8086
   8087:8087
   1455:1455
   19876-19880:19876-19880
   ```
6. **存储卷映射**（可选）：
   ```
   /Users/zmy/Desktop/AIClient/configs:/app/configs
   /Users/zmy/Desktop/AIClient/logs:/app/logs
   ```
7. 点击「创建」

### 步骤 5：访问服务

浏览器打开：`http://你的服务器IP:3000`

**无需密码，点击一键登录即可。**

## 方式二：Docker Compose 部署（推荐）

### 步骤 1：进入项目目录

```bash
cd ~/Desktop/AIClient
```

### 步骤 2：构建镜像

```bash
docker compose build
```

### 步骤 3：启动服务

```bash
docker compose up -d
```

### 步骤 4：查看状态

```bash
# 查看容器状态
docker compose ps

# 查看日志
docker compose logs -f
```

### 步骤 5：访问服务

浏览器打开：`http://localhost:3000`

## 方式三：纯 Docker 命令

### 构建镜像

```bash
cd ~/Desktop/AIClient
docker build -t aiclient2api:latest .
```

### 运行容器

```bash
docker run -d \
  --name aiclient2api \
  --restart unless-stopped \
  -p 3000:3000 \
  -p 8085-8087:8085-8087 \
  -p 1455:1455 \
  -p 19876-19880:19876-19880 \
  -v $(pwd)/configs:/app/configs \
  -v $(pwd)/logs:/app/logs \
  aiclient2api:latest
```

## 🚨 常见问题

### 1. 端口被占用

**错误信息**：`Bind for 0.0.0.0:3000 failed: port is already allocated`

**解决方案**：
```bash
# 查看占用端口的进程
lsof -i :3000

# 停止占用端口的进程
kill -9 <PID>
```

### 2. 构建失败

**可能原因**：
- 网络连接问题
- Docker 版本过旧
- 磁盘空间不足

**解决方案**：
```bash
# 检查 Docker 版本
docker --version

# 清理磁盘空间
docker system prune -a

# 重试构建
docker compose build
```

### 3. 无法访问服务

**检查步骤**：
```bash
# 1. 检查容器状态
docker compose ps

# 2. 查看容器日志
docker compose logs aiclient2api

# 3. 检查防火墙
sudo ufw status

# 4. 测试本地访问
curl http://localhost:3000
```

### 4. 1Panel 构建超时

**解决方案**：
- 增加构建超时时间（在 1Panel 设置中）
- 使用国内 Docker 镜像源
- 检查网络连接
